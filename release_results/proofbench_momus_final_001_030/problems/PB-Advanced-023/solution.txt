**The Final Blueprint:**

**Problem:**
On a table of size $3002 \times 3001$, a stone is placed on $(1, 1)$. Peter selects 3000 traps $T$ such that each row $r \in \{2, \dots, 3001\}$ has exactly one trap and each column $c \in \{1, \dots, 3001\}$ has at most one. James moves to adjacent cells. Entering a trap reveals it, resets the stone to $(1, 1)$, and costs 1 point. Find the smallest $n$ such that James reaches row 3002 before $n$ points.

**I. The Upper Bound ($n \le 3001$)**
We prove James can reach row 3002 with at most 3000 points.
1.  **Existence of a Safe Column:** Let $T$ be the set of 3000 traps chosen by Peter. Let $C = \{1, \dots, 3001\}$ be the set of columns. Since $|T| = 3000$ and each column contains at most one trap, the number of columns containing a trap is at most 3000. By the Pigeonhole Principle, there exists at least one column $j^* \in C$ such that no cell $(r, j^*)$ is in $T$ for any $r$.
2.  **James's Strategy:** James attempts to walk down columns $j = 1, 2, \dots, 3001$ sequentially. For each $j$, he moves horizontally in Row 1 to $(1, j)$ and then moves vertically toward Row 3002.
3.  **Penalty Analysis:** If James hits a trap in column $j$, he receives 1 point and is reset to $(1, 1)$. He then proceeds to attempt column $j+1$. Since each column has at most one trap, James hits at most one trap per column. In the worst-case scenario, he attempts all 3000 columns that contain traps before finally attempting the trap-free column $j^*$.
4.  **Conclusion:** James reaches row 3002 with at most 3000 points. Thus, $n=3001$ is sufficient.

**II. The Lower Bound ($n \ge 3001$)**
We prove Peter can force James to receive 3000 points.
1.  **Adversary Trap Configuration:** Peter selects a trap distribution $\sigma: \{2, \dots, 3001\} \to \{1, \dots, 3000\}$ defined by $\sigma(r) = r-1$. The trap set is $T = \{(r, r-1) : 2 \le r \le 3001\}$. Column 3001 is trap-free.
2.  **Obstruction of Progress:** James starts at $(1, 1)$. To reach row 3002, he must pass through every row $r \in \{2, \dots, 3001\}$.
3.  **The Adjacency and Reset Constraint:**
    *   To cross row 2, James must enter a cell $(2, c_2)$. If $c_2 = 1$, he hits trap $(2, 1)$ and resets.
    *   After resetting, to avoid $(2, 1)$, James must move to row 2 via some column $c_2 \in \{2, \dots, 3001\}$. Once he is at row 2, he must move to row 3.
    *   To cross row 3, he must enter a cell $(3, c_3)$. If $c_3 = 2$, he hits trap $(3, 2)$ and resets.
    *   Crucially, if James tries to avoid $(3, 2)$ by moving horizontally within row 3 from another column (say column 1), he must have first reached row 3 in column 1. However, to reach row 3 in column 1, he must have passed $(2, 1)$, which is a trap.
4.  **Information Bottleneck:** James can only know a trap's location by hitting it. Peter's "staircase" arrangement ensures that any column $c$ that is "safe" for row $r$ (because its trap is in a different row) can only be reached by James if he has already successfully navigated all rows $k < r$.
5.  **Inductive Penalty:** For each $k \in \{1, \dots, 3000\}$, to "learn" that a column $c_k$ is safe for all rows, James must first attempt to cross row $k+1$. Peter’s construction ensures that the only way to bypass row $k+1$ is to use a column $c > k$. Every such attempt for a new row $k+1$ using a previously untested column $c=k$ results in a trap hit.
6.  **Conclusion:** James must hit at least one trap for each of the 3000 trap-containing rows to identify a fully safe path to row 3002. Thus, Peter can force 3000 points.

**III. Final Result**
James can guarantee reaching the goal with 3000 points, and Peter can force 3000 points. The smallest integer $n$ such that James reaches the goal before receiving $n$ points is $3000 + 1 = 3001$.

**Final Answer:** 3001
